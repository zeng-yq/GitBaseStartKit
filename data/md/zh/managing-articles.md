# GitBase 管理面板中的文章管理

GitBase 提供了一个强大且用户友好的界面来管理您网站的文章。本指南将引导您使用管理面板完成查看、编辑和创建新文章的过程。

## 访问文章管理部分

1. 通过导航到 `https://yourgitbasesite.com/admin` 并输入您的访问密码来登录管理面板。
2. 在管理仪表板上，找到并点击"Manage Articles"或"Articles"选项。

## 查看文章

1. 在文章管理部分，您将看到所有现有文章的列表。
2. 每篇文章条目通常显示：
   - 标题
   - 简短描述或摘录
   - 创建日期
   - 最后修改日期

## 编辑现有文章

1. 在列表中找到您要编辑的文章。
2. 点击文章标题旁边的"Edit"按钮。
3. 您将被带到文章编辑器页面，可以在那里修改：
   - 标题：您文章的主要标题
   - 描述：简短摘要或副标题
   - 内容：您文章的主体（Markdown 格式）
   - Slug：标题的 URL 友好版本（如果可编辑）
4. 在提供的字段中做出所需的更改。
5. 使用内容区域的 Markdown 编辑器。您通常可以：
   - 格式化文本（粗体、斜体等）
   - 添加标题
   - 插入链接和图片
   - 创建列表
6. 如果选项可用，预览您的更改。
7. 满意后，点击"Save"或"Update"按钮。

## 创建新文章

1. 在文章管理部分，查找"New Article"或"Create Article"按钮。
2. 点击此按钮打开文章创建表单。
3. 填写以下字段：
   - 标题：为您的新文章输入一个引人注目的标题
   - 描述：撰写简短摘要或副标题
   - 内容：使用 Markdown 撰写您的文章内容
   - Slug：如果需要，输入您标题的 URL 友好版本
4. 使用 Markdown 编辑器根据需要格式化您的内容。
5. 如果可用，使用预览功能查看您文章的外观。
6. 当您对新文章满意时，点击"Create"或"Publish"。

## 理解 Markdown

GitBase 使用 Markdown 进行文章格式化。以下是一些基本的 Markdown 提示：

- 使用 `#` 表示标题（例如，`# 主标题`、`## 副标题`）
- 用 `*` 包裹文本表示斜体，用 `**` 表示粗体
- 用 `[链接文本](URL)` 创建链接
- 用 `![替代文本](图片 URL)` 插入图片

## 管理文章元数据

根据您的 GitBase 设置，您可能能够管理文章的附加元数据：

- 标签或类别
- 特色图片
- 发布日期
- 作者信息

在文章编辑器中查找这些选项并根据需要填写。

## 与 GitHub 同步

保存或创建文章后：

1. GitBase 将自动将您的更改与配置的 GitHub 仓库同步。
2. 在仓库的 `data/md/` 目录中将创建（对于新文章）或更新（对于编辑）一个新的 Markdown 文件。
3. `data/json/` 目录中的 `articles.json` 文件也将被更新以反映更改。

## 最佳实践

1. **定期保存**：频繁保存您的工作以防止内容丢失。
2. **一致的格式**：在您的文章中保持一致的样式以获得专业的外观。
3. **优化的标题和描述**：撰写清晰、对 SEO 友好的标题和描述。
4. **图片优化**：如果包含图片，确保它们已针对 Web 使用进行优化。
5. **校对**：在发布前始终校对您的文章。

## 故障排除

如果您在管理文章时遇到问题：

1. **更改未保存**：检查您的网络连接并重试。如果问题持续存在，请尝试刷新页面并重新输入您的更改。
2. **格式问题**：如果您的 Markdown 渲染不正确，请仔细检查您的语法。
3. **同步错误**：如果更改未在 GitHub 中反映，请验证您的 GitHub 令牌权限和仓库设置。

通过遵循这些指南，您应该能够通过管理面板有效地管理 GitBase 网站上的文章。请记住，您的所有内容都通过 GitHub 进行版本控制，让您可以轻松跟踪更改并在必要时回滚。
