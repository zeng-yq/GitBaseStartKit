## 我如何贡献？

### 报告错误

本节将指导您如何为 GitBase 提交错误报告。遵循这些指南有助于维护者和社区理解您的报告、重现行为并找到相关报告。

**提交错误报告之前**

* 查看文档中常见问题和问题列表。
* 确保错误尚未被报告，请在 GitHub 上搜索 [Issues](https://github.com/yourusername/gitbase/issues)。

**我如何提交（好的）错误报告？**

错误被跟踪为 [GitHub issues](https://guides.github.com/features/issues/)。创建一个 issue 并提供以下信息：

* 使用清晰且描述性的标题来识别问题。
* 尽可能详细地描述重现问题的确切步骤。
* 提供具体示例来演示这些步骤。

### 建议增强功能

本节将指导您如何为 GitBase 提交增强建议，包括全新的功能和现有功能的小幅改进。

**提交增强建议之前**

* 检查该增强功能是否已被建议。
* 确定应该在哪个仓库中建议该增强功能。

**我如何提交（好的）增强建议？**

增强建议被跟踪为 [GitHub issues](https://guides.github.com/features/issues/)。创建一个 issue 并提供以下信息：

* 使用清晰且描述性的标题来识别建议。
* 尽可能详细地提供建议增强功能的分步描述。
* 提供具体示例来演示这些步骤。

### 您的第一次代码贡献

不确定从哪里开始为 GitBase 做出贡献？您可以从查看这些 `beginner` 和 `help-wanted` issue 开始：

* [Beginner issues](https://github.com/yourusername/gitbase/labels/beginner) - 应该只需要几行代码和一两个测试的 issue
* [Help wanted issues](https://github.com/yourusername/gitbase/labels/help%20wanted) - 应该比 `beginner` issue 更复杂一些的 issue

### 拉取请求

此处描述的过程有几个目标：

- 维护 GitBase 的质量
- 修复对用户重要的问题
- 让社区参与共同努力，打造最好的 GitBase
- 为 GitBase 的维护者提供一个可持续的系统来审查贡献

请遵循以下步骤，让您的贡献被维护者考虑：

1. 遵循[模板](PULL_REQUEST_TEMPLATE.md)中的所有说明
2. 遵循[样式指南](#styleguides)
3. 提交拉取请求后，验证所有[状态检查](https://help.github.com/articles/about-status-checks/)是否通过

## 样式指南

### Git 提交消息

* 使用现在时（"Add feature" 而不是 "Added feature"）
* 使用命令式（"Move cursor to..." 而不是 "Moves cursor to..."）
* 将第一行限制在 72 个字符或更少
* 在第一行之后自由引用 issue 和拉取请求

### JavaScript 样式指南

所有 JavaScript 必须遵守 [JavaScript Standard Style](https://standardjs.com/)。

### 文档样式指南

* 使用 [Markdown](https://daringfireball.net/projects/markdown/)。
