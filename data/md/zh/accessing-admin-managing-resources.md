# GitBase 中访问管理面板和管理资源

GitBase 提供了一个用户友好的管理界面来管理您网站的资源。本指南将引导您完成访问管理面板和有效管理资源的过程。

## 访问管理面板

1. **导航到管理 URL**：打开您的网络浏览器，访问您的 GitBase 网站 URL，后跟 `/admin`。例如：`https://yourgitbasesite.com/admin`。

2. **输入访问密码**：系统会提示您输入访问密码。这是您在环境变量中设置的 `ACCESS_PASSWORD`。输入后点击"Login"。

3. **管理仪表板**：成功通过身份验证后，您将被定向到管理仪表板。在这里，您将看到各种管理选项，包括"Manage Resources"。

## 管理资源

### 查看资源

1. 从管理仪表板中，点击"Manage Resources"。
2. 您将看到所有当前资源的列表，显示它们的名称、描述和 URL。

### 添加新资源

1. 在资源管理页面中，找到"Add New Resource"部分，通常在资源列表的底部。
2. 填写以下字段：
   - **Name**：输入资源的名称。
   - **Description**：提供资源的简要描述。
   - **URL**：输入资源的完整 URL。
3. 点击"Add New"按钮保存资源。
4. 新资源将出现在列表中，更改将自动与您的 GitHub 仓库同步。

### 修改现有资源

1. 在资源列表中找到您要编辑的资源。
2. 点击资源旁边的"Edit"按钮。
3. 资源的字段将变为可编辑状态。
4. 对名称、描述或 URL 进行所需的更改。
5. 点击"Save"按钮确认您的更改。
6. 更新的资源信息将显示在列表中并与 GitHub 同步。

### 删除资源

注意：当前版本的 GitBase 可能不支持通过管理界面直接删除。如果您需要删除资源：

1. 编辑资源。
2. 将其内容替换为占位符文本或在描述中标记为"已弃用"。
3. 保存更改。
4. 要永久删除，您可能需要手动从 GitHub 仓库的 `resources.json` 文件中删除该条目。

## 同步更改

GitBase 会自动将您的更改与您环境变量中配置的 GitHub 仓库同步。进行更改后：

1. GitHub 仓库中的 `resources.json` 文件将被更新。
2. 这些更改将在短暂的延迟（通常几分钟，取决于您的托管设置）后在您的实时网站上反映出来。

## 最佳实践

1. **定期备份**：虽然 GitBase 使用 GitHub 进行版本控制，但定期备份您的 `resources.json` 文件是一个好习惯。
2. **描述性名称和描述**：使用清晰、简洁的名称和描述来使您的资源易于理解和查找。
3. **URL 验证**：请始终仔细检查您输入的 URL，确保它们正确且有效。
4. **一致的格式**：在您的资源描述中保持一致的样式以获得专业的外观。

## 故障排除

如果您在管理资源时遇到问题：

1. **更改未显示**：确保您的 GitHub 令牌具有必要的权限，并检查您的 GitHub 仓库中提交历史记录中的任何错误消息。
2. **无法保存**：验证您的网络连接并尝试在尝试保存之前刷新页面。
3. **访问被拒绝**：仔细检查您是否使用了正确的 `ACCESS_PASSWORD`，并且它与您环境变量中的密码匹配。

通过遵循这些步骤，您应该能够通过管理面板有效地管理 GitBase 网站上的资源。请记住，所有更改都通过 GitHub 进行版本控制，让您可以跟踪修改并在必要时回滚。
